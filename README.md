# 🗺️ 自分専用飲食店・観光マップ (GitHub直結動画対応版)

スプレッドシートをデータベースとして活用し、重い動画や写真ファイルはGitHubリポジトリ内に直接格納して管理する、自分専用の軽量・高速マップアプリケーションです。

---

## 🚀 クイックスタート

### 1. 初回設定（GitHubアクセストークンの登録）
1. アプリ（Webサイト）を開きます。
2. ［+登録］ボタン（または既存店舗の「編集」）を押し、登録画面の下部にある **「🔑 トークンを設定・変更する」** ボタンをクリックします。
3. ご自身のGitHubの **個人アクセストークン（Classic PAT / repo権限が必要）** を入力して保存します。
   * ※トークンはブラウザ（LocalStorage）に安全に保存され、外部に送信されません。

---

## 🎬 動画・写真のアップロード運用手順

アプリの動作を軽く保ち、スマホのクラッシュを防ぐため、動画や写真の追加は以下の手順で行います。

### 【手順1】アプリに店舗を登録する
1. アプリ上の「［+登録］」から、店名や住所を入力して保存します。
2. **重要:** ここで決めた「店名」がそのままGitHub上のフォルダ名になります。

### 【手順2】GitHubに直接ファイルを格納する
GitHubのWeb画面、またはGitHub Desktopなどのツールを使い、以下のルールに従ってフォルダを作成し、動画（.mp4 / .mov 等）や写真ファイルを直接アップロードします。

* **飲食店のマッピングの場合:**
  `shops/Restaurant/【アプリに登録した正確な店名】/`
  (例: `shops/Restaurant/寿花/movie1.mp4`)
* **その他マップ（観光地など）の場合:**
  `shops/etc/【アプリに登録した正確な店名】/`
  (例: `shops/etc/高千穂峡/photo1.jpg`)

### 【手順3】アプリ側でメモ（キャプション）を付ける
1. ファイルのアップロード完了後、アプリで対象店舗の「編集」画面を開きます。
2. 店名の下に **「✅ GitHub上にフォルダ確認: 計 〇 件のファイルを検出しました」** と自動で表示されます。
3. 自動検出されたファイル名の一覧の横に、それぞれの「動画・写真のメモ」を入力して「登録（保存）」ボタンを押します。
4. 最後にヘッダーの **「🐙 GitHubへJSON保存」** を押して、アプリ全体のデータを確定させます。

## 💡 知っておくと便利な仕様・注意点

### ⚡ ギガ節約＆クラッシュ防止（遅延読み込み仕様）
店舗詳細画面を開いた際、動画データそのものは自動でダウンロードされません。
画面上の **「▶ タップして動画を読み込む」** をクリックした瞬間に初めてGitHubから動画が読み込まれるため、どれだけ重い動画（数百MBなど）を置いてもページが一瞬で開き、通信量を大幅に節約できます。

### 🍏 iPhone撮影の動画 (.mov) について
* iPhoneで撮影した `.mov` 形式もそのままアップロード可能です。
* ただし、Androidや一部のPCブラウザでは `.mov` の直接再生に対応していないため、非対応ブラウザで開いた場合は自動的に **「📥 ダウンロード / 外部再生」** ボタンに切り替わる安心設計になっています。
* 全ての端末でスムーズに再生させたい場合は、`.mp4` 形式でのアップロードを推奨します。

### 🔄 店名を変更する場合の注意
アプリ上で後から「店名」を変更した場合、GitHub側にあるフォルダ名（例: `寿花`）も手動で新しい店名に変更してください。名前が一致していないと、アプリ側から動画が一時的に見えなくなります（フォルダ名を合わせれば復活します）。

## 🛠️ システム構成・連携先
* **フロントエンド:** `index.html` (Vanilla JS / CSS)
* **データベース:** Googleスプレッドシート (Google Apps Script API 経由)
* **メディアストレージ & 構成管理:** GitHub Repository (`shopdata.json` / `shops/` フォルダ)
